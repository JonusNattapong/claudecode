Configuration

Config Files

Claude Code uses JSON configuration files:

  • settings.json - Global settings (~/.claude/settings.json)
  • settings.local.json - Local overrides (gitignored)
  • .claude/settings.json - Project settings
  • .env - Environment variables

Basic Settings

{
  "model": "claude-sonnet-4-20250514",
  "provider": "anthropic",
  "theme": "dark"
}

Key Options

Model & Provider

"model": "claude-sonnet-4-20250514",
"provider": "anthropic"

See Providers for available models.

Theme

"theme": "dark"      // dark, light, system

Permissions

"permissions": {
  "mode": "auto"     // auto, ask-first, bypass
}

Custom Model (Other Provider)

"model": "openai/gpt-4o",
"provider": "openai"

MCP Servers

"mcpServers": {
  "filesystem": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"]
  }
}

Agents

"agentIds": {
  "default": "general-purpose"
}

Environment Variables

# Required API Keys
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
GOOGLE_API_KEY=...
OPENROUTER_API_KEY=sk-or-...

# Model Override
ANTHROPIC_MODEL=claude-sonnet-4-20250514

# Ollama (Local)
OLLAMA_HOST=http://localhost:11434

# Other Providers
XAI_API_KEY=...
MISTRAL_API_KEY=...
GITHUB_TOKEN=...

Interactive Config

Tip: Use /config command to edit settings interactively.

YOLO Modes

Claude Code supports progressive permission bypass:

ModeDescription
yoloLiteYOLO Lite: Auto-allow most actions with guardian checks
yoloYOLO ALLOW: Broad auto-approval for trusted sessions
yoloMaxYOLO MAX: Full autonomy + sandbox bypass when available
 
/yolo              # Open YOLO mode picker
/yolo yolo         # Directly switch
/yolo default      # Back to ask-first
