Context Compaction

How Claude Code manages context window size through intelligent compaction.

Why Compaction?

AI models have limited context windows (e.g., 200K tokens). When conversations get long, Claude must either:

  • Truncate old messages (lose context)
  • Summarize/compact messages (retain key info)

How Compaction Works

// src/services/compact/compact.ts

async function compactContext(messages: Message[]) {
  // 1. Analyze current context
  const tokenCount = await countTokens(messages)
  const threshold = getContextThreshold()

  // 2. If over threshold, compact
  if (tokenCount > threshold) {
    // 3. Identify messages to preserve
    const preserved = identifyPreservedMessages(messages)

    // 4. Summarize messages to remove
    const summarized = await summarizeMessages(
      messages.filter(m => !preserved.includes(m))
    )

    // 5. Replace original with summarized version
    return buildCompactedContext(preserved, summarized)
  }

  return messages
}

Preserved Content

Some content is protected from compaction:

  • System prompts — Instructions always kept
  • Memory references — Memory-loaded content
  • Recent messages — Last N messages preserved
  • Tool schemas — Tool definitions kept
  • Attachments — File context preserved

Compaction Triggers

// Auto-compact when context exceeds threshold
const CONTEXT_THRESHOLD = 0.7 // 70% of max

// Compact when approaching limit
if (tokenCount / maxTokens > CONTEXT_THRESHOLD) {
  await triggerCompaction()
}

Key Learnings

  1. Proactive not reactive — Compacts before hitting limit
  2. Preserve important context — System prompts, memories protected
  3. Multi-level compaction — Micro to major based on need
  4. Preserve recent — Last messages always kept intact
  5. User control — /compact command for manual trigger
  6. Token tracking — Accurate token counting crucial
