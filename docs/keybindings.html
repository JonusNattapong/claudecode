Keybindings

Keyboard shortcuts for navigation, editing, and session control.

Default Shortcuts

KeyAction
EnterSubmit prompt / execute
Shift+EnterNew line in input
TabAccept ghost text / autocomplete suggestion
Up / DownNavigate command history
Ctrl+CCancel current operation
Ctrl+DExit / close session
Ctrl+LClear terminal screen
Ctrl+RSearch command history (across all projects)
Shift+TabCycle permission modes
EscapeClose sidebar / cancel

Permission Mode Cycling

Shift+Tab cycles through permission modes:

  1. Default — ask before every tool execution
  2. Auto — automatically approve safe operations
  3. Plan — only plan, no execution
  4. YOLO Lite — auto-approve most operations
  5. YOLO — full autonomous mode
  6. YOLO MAX — unrestricted autonomous mode

See Permission Model for details on each mode.

Vim Mode

Clew includes a Vim-inspired navigation mode for efficient keyboard-driven interaction.

KeyVim Mode Action
j / kScroll down / up through output
g / GGo to top / bottom of output
/Search within output
n / NNext / previous search match
iEnter insert mode (back to input)
qQuit Vim mode

Toggle Vim mode with /vim or configure it in settings.

Custom Keybindings

Create a keybindings.json file to customize or add shortcuts:

{
  "keybindings": [
    {
      "key": "ctrl+p",
      "command": "model",
      "description": "Quick model switch"
    },
    {
      "key": "ctrl+shift+f",
      "command": "compact",
      "description": "Compact context"
    }
  ]
}

Keybinding files are loaded from ~/.claude/keybindings.json (global) and .claude/keybindings.json (project-level). The key field supports modifiers (ctrl, shift, alt, meta) and a single character or named key.

Terminal Configuration

Some shortcuts require terminal-specific setup:

  • Shift+Enter for new lines — may need to configure your terminal to not capture it
  • Ctrl+R for history search — works in most modern terminals
  • Custom keybindings — require a terminal that passes the key combination through
