Agent System

How Claude Code manages subagents for parallel task execution and background sessions via the Supervisor daemon.

Overview

Agents are sub-Claudes that run in the background, allowing parallel task execution. Each agent has its own conversation, state, and can work independently. The system consists of three layers:

  1. AgentTool — The tool the AI model calls to spawn subagents
  2. LocalAgentTask — Manages agent state, lifecycle, and UI display
  3. Supervisor Daemon — Hosts background sessions as independent processes

Creating an Agent

Via Command

/agent Write tests for the auth module

Via Custom Agent Definition

Define a custom agent in .claude/agents/:

# .claude/agents/test-runner.md
---
name: test-runner
description: Run tests and report results
model: claude-sonnet-4-7
tools: Bash, Read, Edit
---

# Test Runner Agent

You are a testing specialist.

## Your Task
1. Understand the code to test
2. Write appropriate tests
3. Run tests
4. Report results

## Output Format
- Tests written
- Test results
- Any failures

Agent Lifecycle

1. Create agent with task description
   ↓
2. Initialize agent state (taskId, status, messages, agentType)
   ↓
3. Spawn agent process (separate conversation)
   ├── Background: registerAsyncAgent() — immediate return
   └── Foreground: registerAgentForeground() — waits for backgrounding
   ↓
4. Agent runs in background
   ├── Progress updates via agent.onProgress()
   └── Tool count, tokens, summary updated
   ↓
5. Complete / Fail
   ├── agent.onComplete(result) — report results
   └── Move to Completed / Failed state
   ↓
6. Cleanup (remove from state, free resources)

Background vs Foreground

// Background agent (immediate return)
const agent = registerAsyncAgent({
  prompt: "Write tests for the auth module",
  agentType: "general-purpose",
  onProgress: (progress) => { /* update UI */ },
  onComplete: (result) => { /* report results */ }
})

// Foreground agent (waits for backgrounding)
const { taskId, backgroundSignal } = registerAgentForeground({
  prompt: "Investigate the flaky test",
  agentType: "verification"
})
// Can continue main conversation while agent runs

Agent Communication

// Progress updates
agent.onProgress = (progress) => {
  // Update UI with tool count, tokens, summary
  // progress.toolsUsed: number of tools executed
  // progress.tokensUsed: token count
  // progress.summary: AI-generated summary
}

// Completion
agent.onComplete = (result) => {
  // Report results back to main conversation
  // result.output: final output text
  // result.success: boolean
}

Supervisor Daemon

The supervisor is a per-user background process that hosts independent session processes:

Architecture

  • One supervisor per user, auto-started on first background session
  • Communicates via Windows named pipe or Unix socket (IPC)
  • Each background session is a child Bun process
  • Roster persisted to ~/.claude/daemon/roster.json
  • Auto-exits when all sessions finished and no terminal is connected for ~1 hour
  • Watches the installed Claude Code binary and restarts into the new version after auto-update

IPC Protocol

JSON, newline-delimited over pipe/socket:

Request:  {"type":"<command>","sessionId?":"...","cwd?":"...","prompt?":"..."}
Response: {"ok":true,"data":{...}} or {"ok":false,"error":"..."}

Commands: spawn, attach, stop, respawn, rm, list, logs, shutdown, ping

State Storage

PathContents
~/.claude/daemon.logSupervisor log
~/.claude/daemon/roster.jsonList of running background sessions
~/.claude/jobs/<id>/state.jsonPer-session state

Agent View

Agent view is the terminal UI for managing multiple background sessions. Open with claude agents.

Components

  • AgentsMenu — Main menu container
  • AgentViewDashboard — Session list grouped by state
  • AgentViewRow — Individual session row with icon, name, summary, PR status
  • AgentViewPeekPanel — Peek panel showing recent output
  • AgentViewDispatchInput — Input for dispatching new sessions
  • AgentViewShortcutsHelp — Keyboard shortcuts overlay

AI Row Summaries

Each row has an AI-generated one-line summary powered by useAgentViewSummaries. Summaries refresh at most once every 15 seconds while a session is actively working.

PR Status Detection

The AgentPRStatus service detects when a session opens a pull request and shows a colored status dot on the row.

Custom Agents

Custom agents are loaded from .claude/agents/ directory. Each agent is a markdown file with frontmatter:

---
name: my-agent
description: What this agent does
model: claude-sonnet-4-7
tools: Bash, Read, Edit, Write
isolation: worktree
---

# My Agent

You are a specialized agent for...

Supported Frontmatter Fields

FieldDescription
nameAgent identifier
descriptionWhat the agent does (shown in menus)
modelOverride model for this agent
toolsTools available to this agent
isolationworktree to run in isolated git worktree
permissionModePermission mode for dispatched sessions

Key Learnings

  1. Parallel execution — Multiple agents run simultaneously without blocking the main session
  2. State isolation — Each agent has its own conversation, state, and resources
  3. Progress tracking — Real-time progress updates with tool count, tokens, and AI summaries
  4. Background/foreground — Flexible execution modes for different workflows
  5. Custom definitions — Users can define specialized agents with markdown frontmatter
  6. Resource management — Proper cleanup on completion, supervisor auto-exits when idle
  7. IPC architecture — Supervisor daemon enables detached sessions that survive terminal close
  8. Worktree isolation — Parallel sessions write to separate git worktrees
  9. AI summaries — Haiku-class model generates concise row summaries
  10. PR status tracking — Automatic detection and display of pull request status
