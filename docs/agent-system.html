Agent System

How Claude Code manages subagents for parallel task execution.

What are Agents?

Agents are sub-Claudes that run in the background, allowing parallel task execution. Each agent has its own conversation and can work independently.

Agent Types

TypeDescription
general-purposeGeneral coding tasks
exploreExplore codebase
planPlan complex tasks
verificationVerify code correctness
customUser-defined agents

Creating an Agent

// Use /agent command
/agent Write tests for the auth module

Agent Lifecycle

1. Create agent with task description
2. Initialize agent state (taskId, status, messages)
3. Spawn agent process (separate conversation)
4. Agent runs in background
5. Progress updates via callbacks
6. Complete/fail → report results
7. Cleanup (remove from state)

Background vs Foreground

// Background agent (immediate return)
const agent = registerAsyncAgent({...})

// Foreground agent (waits for backgrounding)
const { taskId, backgroundSignal } = registerAgentForeground({...})
// Can continue main conversation while agent runs

Agent Communication

// Progress updates
agent.onProgress = (progress) => {
  // Update UI with tool count, tokens, summary
}

// Completion
agent.onComplete = (result) => {
  // Report results back to main conversation
}

Custom Agents

// Define custom agent in .claude/agents/
// .claude/agents/test-runner.md
---
name: test-runner
description: Run tests and report results
---

# Test Runner Agent

You are a testing specialist.

## Your Task
1. Understand the code to test
2. Write appropriate tests
3. Run tests
4. Report results

## Output Format
- Tests written
- Test results
- Any failures

Key Learnings

  1. Parallel execution — Multiple agents run simultaneously
  2. State isolation — Each agent has own conversation
  3. Progress tracking — Real-time progress updates
  4. Background/foreground — Flexible execution modes
  5. Custom definitions — User can define specialized agents
  6. Resource management — Proper cleanup on completion
