MCP Integration

How Claude Code integrates with Model Context Protocol servers.

What is MCP?

Model Context Protocol (MCP) is a standardized way to connect AI models to external tools and data sources.

MCP Files

src/services/mcp/
├── client.ts                  # MCP client
├── types.ts                   # Type definitions
├── config.ts                  # Configuration
├── utils.ts                   # Utilities
├── auth.ts                    # Authentication

MCP Configuration

// settings.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"]
    }
  }
}

Loading MCP Servers

// src/utils/plugins/mcpPluginIntegration.ts

async function loadMcpServers() {
  // 1. Get MCP config from settings
  const config = getMcpConfig()

  // 2. For each server, spawn process
  for (const [name, options] of Object.entries(config)) {
    // 3. Connect via stdio/http
    const client = await createMcpClient(options)

    // 4. Get tools from server
    const tools = await client.listTools()

    // 5. Register as Claude tools
    registerMcpTools(name, tools)
  }
}

MCP in Plugins

// Plugin can define MCP servers
{

      "command": "node",
      "args": ["./mcp-server.js"]
    }
  }
}

Key Learnings

  1. Standard protocol — MCP provides universal tool interface
  2. Process isolation — Servers run as separate processes
  3. Tool conversion — MCP tools converted to Claude tools
  4. Config-driven — Servers defined in settings
  5. Plugin integration — Plugins can provide MCP servers
