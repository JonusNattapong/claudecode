Agent Teams & Sub-agents

Orchestrate multiple AI agents working in parallel or sequence — each with its own model, tools, and workspace.

Overview

Clew supports multi-agent coordination through two complementary systems:

  • Agent Runtime (src/agentRuntime/) — Local-first orchestration engine for spawning, checkpointing, and executing complex multi-agent workflows with durable state.
  • Agent View (/agents) — Interactive TUI dashboard for managing background sessions, dispatching tasks, and monitoring agent activity.

Agent Runtime

The agent runtime is a durable orchestration engine stored in src/agentRuntime/:

Core Components

ModuleRole
orchestrator.tsDrives agents through state graph transitions, checkpointing state to disk, handling manual/automatic pause/resume, and routing handoffs between agent profiles.
runStore.tsSaves and loads execution runs locally (.claude/runs/) with secret scrubbing rules to prevent API key leakage into diagnostic logs.
toolGateway.tsMulti-permission capability gateway validating sandbox access and halting guarded shell commands for human approval.
agentRegistry.tsMarkdown frontmatter parser for .claude/agents/ configurations.
workflowRegistry.tsYAML validator for .claude/workflows/ DAG pipelines.
reportBuilder.tsGenerates standardized markdown summaries of agent steps, file changes, and test validations.

Agent View Dashboard

Open the interactive agent dashboard with /agents:

/agents                  # Open agent view dashboard
/agents --cwd /project   # Open with specific working directory
/agents --config         # Open configuration manager
/agents --json           # Machine-readable output

The dashboard provides:

  • Running/Library tabs — Switch between active sessions and saved agent configurations.
  • Dispatch input — Type / for autocomplete suggestions from slash commands, bundled skills, project skills, and plugins. Each suggestion shows its source label ([cmd], [bundled], [skill], [plugin]).
  • PR column — Shows linked PRs per session (PR #123 or 3 PRs).
  • Running indicator — ● N badge showing count of active agent sessions.

Sub-agent Configuration

Define custom sub-agents in .claude/agents/ with Markdown frontmatter:

---
name: code-reviewer
model: claude-sonnet-4-20250514
description: Specialized code review agent
tools:
  - Read
  - Grep
  - Glob
allowedPaths:
  - src/
systemPrompt: |
  You are a code review specialist. Focus on correctness,
  security, and adherence to project conventions.
---

Agent Spec Fields

FieldDescription
nameUnique agent identifier (used in @-mentions).
modelModel to use for this agent. Falls back to session model.
descriptionShown in typeahead suggestions and agent lists.
toolsWhitelist of tools this agent can use. Omit for full access.
allowedPathsRestrict file system access to specific directories.
systemPromptCustom system prompt for this agent.

Workflows

Define multi-agent DAG pipelines in .claude/workflows/ using YAML:

name: pr-review-pipeline
steps:
  - id: lint
    agent: linter
    prompt: "Run linting on the changed files"
  - id: review
    agent: code-reviewer
    prompt: "Review the diff for correctness and security"
    depends_on: [lint]
  - id: test
    agent: test-runner
    prompt: "Run the test suite"
    depends_on: [lint]
  - id: summary
    agent: report-builder
    prompt: "Compile findings into a review summary"
    depends_on: [review, test]

Agent Team Coordination

The orchestrator manages agent execution with these guarantees:

  • Checkpointing — State saved to disk after each step; resume after crash or manual pause.
  • Handoffs — Agents can route work to other agent profiles mid-execution.
  • Tool gating — toolGateway.ts enforces per-agent tool permissions and sandbox boundaries.
  • Secret scrubbing — API keys and tokens are stripped from run logs automatically.
  • Subagent isolation — Sub-agents cannot leak working directory changes back to the parent session.

Named Sub-agents in @-mentions

Registered sub-agents appear in @-mention typeahead suggestions. Type @ in the prompt input to see available agents alongside files and commands.

Background Sessions

clew --bg                # Start a background session
clew --bg --name "review" # Named background session
clew ps                  # List background sessions
clew attach              # Attach to a background session
clew stop <id>           # Stop a background session
