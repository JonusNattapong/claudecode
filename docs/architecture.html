Learning Highlights

1. Multi-Provider Pattern

Each provider implements a common interface, enabling seamless switching:

interface Provider {
  streamMsg(messages, options): AsyncIterable
  nonStreamingMsg(messages, options): Promise
  getModels(): Promise
  getToolResultSchema(): JSONSchema
}

2. Tool Permission System

Permission modes control tool execution safety:

  • Default: Ask before dangerous tools
  • Auto: Auto-approve safe tools
  • YOLO: Progressive bypass levels

3. Plugin Architecture

Plugins can extend functionality through:

  • Custom commands
  • Skills (instruction sets)
  • Hooks (pre/post tool execution)
  • Agent definitions

4. Context Management

Claude Code manages context window through:

  • Session Memory: Current conversation
  • Compact: Reduce context size
  • Token Estimation: Track usage
Tip: Explore src/services/ai/ to understand the provider pattern, and src/utils/permissions/ for the security model.
