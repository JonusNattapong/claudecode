Glossary

Key terms and concepts used throughout Clew.

Agent Loop
The core processing cycle: receive user input → build context → send to model → receive response → execute tool calls → repeat. The agent loop continues until the model produces a text-only response or the user interrupts.
See: Agent Teams & Sub-agents
Agent Runtime
The orchestration engine (src/agentRuntime/) that spawns, checkpoints, and executes multi-agent workflows with durable state. Includes orchestrator, run store, tool gateway, and registries.
See: Agent Teams & Sub-agents
Auto-Compact
Automated context window management that compresses conversation history when approaching the model's token limit. Prevents context overflow in long sessions.
See: Context Window
Bridge Mode
WebSocket-based remote collaboration system enabling session sharing, team onboarding, and access from mobile/web clients.
See: Bridge Mode
CLAUDE.md
Project-level instruction files that define repository-specific rules, conventions, and context. Loaded automatically and injected into the system prompt. Supports nested files for subdirectories.
See: Configuration
Compaction
The process of reducing conversation context size to stay within model token limits. Includes KiloCompact (local log/trace snipping and failed-state consolidation) and semantic turn pruning.
Commands: /compact
Context Window
The maximum number of tokens a model can process in a single request. Clew displays current usage as a percentage and a segmented bar showing token distribution across system prompt, tools, rules, chat, and sub-agents.
Commands: /context
Coordinator Mode
A mode where Clew delegates work to sub-agents, each handling a portion of the task. The coordinator splits the problem, spawns workers, collects results, and synthesizes the final output.
See: Agent Teams & Sub-agents
Daemon Mode
Background process that runs 24/7, picks tasks from a queue, spawns worker sessions, monitors execution, and retries on failure. Managed by supervisor with health checks and auto-respawn.
Commands: /daemon start|stop|status|restart
See: Daemon Mode
Dead Letter
A task status indicating the task has exhausted its retry count. Dead-lettered tasks are preserved for manual inspection and can be requeued with /task requeue.
See: Scheduled Tasks
Hooks
Custom scripts that fire at specific points in the execution lifecycle. Hook points: PreToolUse, PostToolUse, PreBash, PostPrompt, PreAcceptEdit, PermissionDenied.
See: Hooks
KiloCompact
Local context compaction engine that collapses verbose directory trees, glob outputs, stack traces, and large text dumps. Consolidates redundant failed tool runs. Applies keyword-based semantic pruning while preserving protected turns.
See: Compaction
MCP (Model Context Protocol)
An open protocol for connecting AI models to external tools, resources, and prompts. Clew supports MCP servers for extended capabilities like database access, API integrations, and custom tools.
Commands: /mcp
See: MCP
Permission Modes
Tiered system controlling when the AI must ask before executing tools. Modes: default, ask, plan, auto, acceptEdits, bypassPermissions, dontAsk.
Commands: /permissions
See: Permission Model
Plugin
Packaged extensions that add commands, agents, skills, hooks, and MCP servers. Loaded from user and bundled plugin locations. Managed through the marketplace system.
Commands: /plugin
See: Plugins
Provider
An AI model API service. Clew supports 27 providers including Anthropic, OpenAI, Google Gemini, DeepSeek, OpenRouter, Ollama, xAI, Mistral, and Groq.
Commands: /provider
See: Providers
Session
A single conversation instance with its own model, provider, context, and history. Sessions can be saved, listed, restored, and shared via bridge mode.
Commands: /resume
Skill
Reusable procedures, checklists, and domain workflows packaged with prompts and tool configurations. Can be user-defined or come from plugins.
Commands: /skill
See: Skills
Sub-agent
An AI agent spawned by the main session to handle a specific subtask. Each sub-agent can have its own model, tools, allowed paths, and system prompt. Sub-agents are isolated — they cannot leak working directory changes to the parent.
See: Agent Teams & Sub-agents
Supervisor
A process manager that monitors the Clew daemon, performs health checks every 30 seconds, and auto-respawns on crash. Also manages inter-process communication between sessions.
See: Daemon Mode
Task Queue
File-backed priority queue (~/.claude/daemon/tasks.json) for autonomous task execution. Supports priorities, scheduling, dependency chains, tags, leasing, retry with exponential backoff, and dead-letter handling.
See: Scheduled Tasks
Tools
Built-in capabilities the AI can invoke: Read, Write, Edit, Glob, Grep, Bash, Web Search, Research, MCP, Agent, Task, Monitor, and more. 65+ tools available.
Commands: /tools
See: Tools
Workflow
A YAML-defined DAG pipeline of agent steps with dependencies. Workflows define multi-step processes where later steps depend on earlier ones completing successfully.
See: Agent Teams & Sub-agents
