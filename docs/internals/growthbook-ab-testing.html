GrowthBook A/B Testing

A/B testing system architecture and implementation.

Overview

GrowthBook is used for feature flagging and A/B testing within Ceph Code. It enables gradual rollouts, experiment tracking, and targeted feature releases based on user segments.

Architecture

The GrowthBook SDK is initialized at startup with a set of feature definitions loaded from the GrowthBook server. Feature flags are evaluated locally with caching to minimize latency. The system supports:

  • Feature Flags — Boolean, string, number, and JSON flag types
  • Experiments — Multi-variant A/B tests with statistical tracking
  • Forced Variations — Override for testing and debugging
  • Sticky Bucketing — Consistent user experience across sessions

Usage in Code

// Check if a feature is enabled
const isEnabled = getFeatureValue_CACHED_MAY_BE_STALE('tengu_plum_vx3', false);

// Use the feature value
if (isEnabled) {
  // New behavior
}

Configuration

GrowthBook is configured via environment variables. The SDK connects to the GrowthBook API at startup to fetch the latest feature definitions, then caches them locally. Feature values can be forced for testing purposes.
