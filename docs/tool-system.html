Tool System

How 40+ built-in tools work.

Tool Definition

src/tools/<ToolName>/tool.ts

Each tool exports:

export const name = 'Read'           // Tool name
export const description = '...'      // For AI context
export const parameters = zodSchema  // Input validation
export const isEnabled = () => true   // Enable check

// Execution
export async function execute(params, context) {
  // Do work
  return { content: 'result' }
}

Tool Execution Flow

1. AI generates tool call (name + parameters)
2. Parse tool call → validate with Zod schema
3. Check permissions (is tool allowed?)
4. Execute tool with params
5. Return result to AI
6. AI continues with tool result

Permission Levels

// tools.ts - Tool definitions include permission levels
export const toolDefinitions = {
  Read: {
    permission: 'read',      // Safe, usually auto-approved
  },
  Edit: {
    permission: 'write',    // Moderate risk
  },
  Bash: {
    permission: 'dangerous', // High risk, always prompt
  },
}

Tool Hooks

src/services/tools/toolHooks.ts

Execute logic before/after tool use:

// Pre-tool hooks
async function preToolUse(toolName, params) {
  // Log, validate, transform
}

// Post-tool hooks
async function postToolUse(toolName, result) {
  // Log, cleanup, notify
}

Key Learnings

  1. Zod validation — Parameter schemas validated before execution
  2. Permission gating — Tools checked against permission mode
  3. Hook system — Extensible pre/post execution logic
  4. Result normalization — Tool results normalized for AI consumption
Pattern: Tools are the bridge between AI decision and system action — careful design prevents misuse.
