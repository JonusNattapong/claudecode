Plugin System

How Claude Code extends functionality through plugins.

Plugin Structure

my-plugin/
├── .claude-plugin/
│   └── plugin.json          # Manifest
├── commands/
│   └── my-command.md        # Custom slash command
├── agents/
│   └── my-agent.md          # Custom agent
├── skills/
│   └── my-skill.md          # Custom skill
├── hooks/
│   └── hooks.json           # Hook definitions
└── settings/
    └── settings.json        # Plugin settings

Plugin Manifest

// .claude-plugin/plugin.json
{
  "name": "my-plugin",
  "description": "My custom plugin",
  "version": "1.0.0",
  "authors": ["Your Name"],
  "homepage": "https://github.com/...",
  "tags": ["productivity", "git"],
  "commands": ["build", "deploy"],
  "agents": ["test-runner"],
  "skills": ["my-skill"],
  "hooks": {
    "PreToolUse": "./hooks/pre-tool.js",
    "PostToolUse": "./hooks/post-tool.js"
  },
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["-y", "@my/mcp-server"]
    }
  }
}

Plugin Loading

// src/utils/plugins/pluginLoader.ts

async function loadPlugins() {
  // 1. Discover plugins from settings
  const plugins = getConfiguredPlugins()

  // 2. Load each plugin manifest
  for (const plugin of plugins) {
    const manifest = await loadPluginManifest(plugin.path)

    // 3. Load components
    const commands = await loadPluginCommands(plugin.path)
    const agents = await loadPluginAgents(plugin.path)
    const hooks = await loadPluginHooks(plugin.path)
    const mcpServers = await loadPluginMcpServers(plugin.path)

    // 4. Register with system
    registerPlugin(manifest, { commands, agents, hooks, mcpServers })
  }
}

Plugin Hooks

// Available hook points
type HookPoint =
  | 'PreToolUse'      // Before tool execution
  | 'PostToolUse'    // After tool execution
  | 'PreBash'        // Before bash command
  | 'PostBash'       // After bash command
  | 'PrePrompt'       // Before building prompt
  | 'PostPrompt'      // After building prompt
  | 'PreAcceptEdit'  // Before accepting edit

Key Learnings

  1. Manifest-based — Each plugin declares its capabilities
  2. Component-based — Commands, agents, skills, hooks, MCP
  3. Lazy loading — Plugins loaded on-demand
  4. Hook system — Extend behavior at key points
  5. Settings support — Plugins can have user config
  6. Marketplace integration — Install from marketplace
