Skill System

How Claude Code provides specialized instruction sets for specific tasks.

What are Skills?

Skills are specialized instruction sets that guide Claude's behavior for specific tasks. They're loaded into the system prompt when activated.

Skill Structure

skill-name/
├── SKILL.md           # Main skill definition
├── references/         # Reference docs
│   └── patterns.md
├── examples/          # Example usage
│   └── example.md
└── scripts/          # Helper scripts (optional)

Skill Definition

# SKILL.md
---
name: verify
description: Verification workflows for code correctness
---

# Verification Skill

Use this skill when asked to verify code correctness.

## When to Use
- Running tests
- Type checking
- Linting
- Security scanning

## Workflow
1. Identify verification steps
2. Run verification tools
3. Report results
4. Fix issues if found

Loading Skills

// src/skills/loadSkillsDir.ts

async function loadSkills() {
  // 1. Load bundled skills
  const bundled = loadBundledSkills()

  // 2. Load project skills
  const project = await loadProjectSkills()

  // 3. Load plugin skills
  const plugin = await loadPluginSkills()

  // 4. Merge and register
  return mergeSkills(bundled, project, plugin)
}

Using Skills

// Activate skill via command
/skill verify    # Activate verify skill
/skill debug     # Activate debug skill
/skill web-search # Activate web search skill

// Skills can be combined
/skill verify /skill debug

Skill Activation

// When skill is activated, instructions added to system prompt

const skillInstructions = `
# Skill: verify

[Skill instructions loaded from SKILL.md]

# End Skill
`

// Added to system prompt
systemPrompt += skillInstructions

Key Learnings

  1. Markdown-based — Skills defined in SKILL.md
  2. Context injection — Loaded into system prompt
  3. Granular control — Can target specific workflows
  4. Composable — Multiple skills can be combined
  5. Bundled + custom — Built-in + user-defined
  6. Project-scoped — Can be project-specific
