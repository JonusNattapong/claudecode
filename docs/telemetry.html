Telemetry & Tracing

Claude Code includes an OpenTelemetry-based observability system for session tracing, LLM request monitoring, tool execution tracking, and performance analysis.

Architecture

The telemetry system is built on OpenTelemetry (OTEL) and provides three signal types:

SignalPurposeOTEL SDK
TracesDistributed tracing of requests across the system@opentelemetry/sdk-trace-base
MetricsNumeric measurements (tokens, latency, counts)@opentelemetry/sdk-metrics
LogsStructured event logging@opentelemetry/sdk-logs

Span Hierarchy

Each user interaction creates a root interaction span that contains nested child spans:

interaction (root)
├── llm_request        # AI model API call
│   ├── agent_id       # Subagent identifier (G32)
│   └── parent_agent_id # Parent agent identifier (G32)
├── tool_call          # Tool execution
│   ├── tool.name      # Tool identifier
│   ├── tool.input     # Truncated input parameters
│   └── tool.result    # Truncated output
├── user_input         # User prompt received
└── compaction         # Context compaction event

Configuration

Enabling Telemetry

Telemetry is controlled via environment variables. Enhanced tracing requires the ENHANCED_TELEMETRY_BETA feature flag.

Trace Exporters

ExporterEnv VariableUse Case
ConsoleOTEL_TRACES_EXPORTER=consoleLocal debugging, development
OTLPOTEL_TRACES_EXPORTER=otlpProduction observability (Jaeger, Grafana, Datadog)
NoneOTEL_TRACES_EXPORTER=noneDisable trace export (default)

OTLP Endpoint Configuration

# OTLP over HTTP (recommended)
export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318
export OTEL_EXPORTER_OTLP_PROTOCOL=http/protobuf

# OTLP over gRPC
export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317
export OTEL_EXPORTER_OTLP_PROTOCOL=grpc

Resource Attributes

export OTEL_RESOURCE_ATTRIBUTES="service.name=claude-code,service.version=2.1.137"

Session Tracing (BETA)

The session tracing module (src/utils/telemetry/sessionTracing.ts) provides a high-level API:

FunctionPurpose
startInteractionSpan()Begin a new user interaction trace
endInteractionSpan()Complete the interaction and export
startLLMRequestSpan()Trace an LLM API call
endLLMRequestSpan()Complete LLM call with response data
startToolSpan()Trace tool execution
endToolSpan()Complete tool span with result

LLM Request Tracing

LLM request spans include detailed attributes for observability:

AttributeDescription
llm.providerAI provider name (anthropic, openai, etc.)
llm.modelModel identifier
llm.request.tokensInput token count
llm.response.tokensOutput token count
llm.streamingWhether streaming was used
agent_idSubagent ID (G32)
parent_agent_idParent subagent ID (G32)
traceparentW3C trace context for distributed tracing

Event Logging

The event system (src/utils/telemetry/events.ts) emits structured OTEL events:

import { logOTelEvent } from 'src/utils/telemetry/events.js'

await logOTelEvent('claude.session.started', {
  provider: 'anthropic',
  model: 'claude-sonnet-4-7',
  mode: 'yolo',
})

Events include automatic attributes:

  • event.name — Event type identifier
  • event.timestamp — ISO 8601 timestamp
  • event.sequence — Monotonically increasing counter
  • service.name / service.version — From resource config
  • host.arch — Platform architecture (from SEMRESATTRS_HOST_ARCH)

Perfetto Tracing

For in-depth performance analysis, Perfetto tracing (src/utils/telemetry/perfettoTracing.ts) captures fine-grained timing data:

  • Interaction-level spans with wall-clock timing
  • LLM request latency breakdowns
  • Tool execution duration tracking
  • User input processing time
Perfetto vs OTEL: Perfetto tracing provides microsecond-level detail for local debugging. OTEL spans are designed for distributed tracing across services. Both can be active simultaneously without conflict.

W3C Trace Context

Claude Code propagates W3C traceparent headers to subprocesses via the TRACEPARENT environment variable in Bash tool subprocesses. This enables end-to-end tracing from the user prompt through tool execution to child processes.

# Injected into every Bash tool subprocess
TRACEPARENT=00-0af7651916cd43dd8448eb211c80319c-b7ad6b7169203331-01

Plugin Telemetry

Plugin lifecycle events are tracked via src/utils/telemetry/pluginTelemetry.ts:

  • Plugin load and initialization timing
  • Hook execution counts and durations
  • Skill activation events (skillLoadedEvent.ts)

BigQuery Export

For production deployments, the BigQuery exporter (src/utils/telemetry/bigqueryExporter.ts) streams telemetry data to Google BigQuery for long-term storage and analysis. Configure via:

export BIGQUERY_PROJECT_ID=my-project
export BIGQUERY_DATASET_ID=claude_code_telemetry

Privacy Controls

Env VariableEffect
OTEL_LOG_USER_PROMPTS=1Include user prompt text in telemetry events
OTEL_LOG_USER_PROMPTS=0Redact user prompts (default: <REDACTED>)
NODE_ENV=testSuppress all telemetry event emission
Privacy Note: User prompts are redacted by default in telemetry. Tool inputs and outputs are truncated to prevent sensitive data leakage. Enable OTEL_LOG_USER_PROMPTS only in controlled environments where full prompt logging is explicitly required.
