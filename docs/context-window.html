Context Window

How Clew manages the context window — visualization, compaction, and caching.

Overview

The context window holds the current conversation history, system prompts, file contents, and tool results. Clew automatically manages context to stay within the model's limits.

Context Compression

/compact reduces context usage by:

  • Log/trace snipping — removes verbose log output while keeping errors and key signals
  • Failed-state consolidation — collapses repeated failed attempts into a summary
  • Semantic turn pruning — removes turns that don't contribute to the current direction
  • Tool result truncation — keeps only the essential parts of large tool outputs
> /compact
Compacting context...
■ Log/trace snipping    — removed 4.2k tokens
■ Failed consolidation  — removed 1.8k tokens
■ Turn pruning          — removed 3.1k tokens
→ Saved 9.1k tokens (52% → 41%)

Prompt Caching

Clew uses prompt caching automatically to reduce latency and costs. Key behaviors:

  • System prompt and project instructions are cached across turns
  • Switching models resets the cache (first turn after switch is slower)
  • CLAUDE.md changes take effect on the next session, not mid-session
  • KiloCompact runs automatically when context exceeds 80%

Best Practices

  • Compact proactively — run /compact before starting a new task in a long session
  • Clear when switching domains — /clear gives a fresh context for unrelated tasks
  • Use specific file paths — narrow reads save context compared to broad globs
  • Monitor with /context — keep an eye on usage, especially during long sessions
