Sessions

Name, save, resume, and manage your Clew conversations.

Clew automatically saves every conversation as a session transcript. You can name sessions for easy recall, resume them later, branch from a checkpoint, or rewind to a previous state.

Session Commands

CommandDescription
/sessionShow current session info — name, ID, start time, context usage
/session shareGenerate a QR code for remote session sharing (bridge mode)
/rename <name>Rename the current conversation
/resumeOpen an interactive picker of saved sessions to resume
/rewindRestore code and/or conversation to a previous point
/branchCreate a branch of the current conversation at this point
/copy [N]Copy the last response to clipboard (or Nth latest)
/exportExport current conversation to a file or clipboard
/recapShow a summary of the session when you return to it
/clearClear conversation history and free up context window

CLI Flags

FlagDescription
-c, --continueContinue the most recent session
-r, --resumeShow interactive picker to resume a saved session
--from-pr <url>Resume the session that created a given PR
# Continue last session
clew -c

# Resume a specific session
clew -r

# Resume session from a PR URL
clew --from-pr https://github.com/org/repo/pull/123

Session Transcripts

Session transcripts are saved to ~/.claude/projects/<project-hash>/transcripts/. Each transcript is a JSON file that stores the full conversation history, tool results, and metadata.

  • Auto-save — transcripts are saved continuously during a session
  • Naming — unnamed sessions get a timestamp-based name; rename with /rename
  • Search — use bun run session list to browse all saved sessions
  • Storage — transcripts persist across machine restarts

Session Branching

Use /branch to create a fork of the current conversation. This saves the entire conversation state up to that point as a new named session, letting you explore alternative approaches without losing context.

> /branch experiment-auth-refactor
> "try a different approach to the auth rewrite"

Rewind

/rewind restores the codebase and/or conversation to an earlier state. You can choose:

  • Code only — revert file changes without losing conversation context
  • Conversation only — restore the conversation to a previous point
  • Both — rewind everything to a checkpoint

Session Bridge

Clew supports remote session sharing via WebSocket bridge. Generate a QR code with /session share to let another developer view your session in real time, or use /bridge to configure persistent remote collaboration.

See Bridge Mode for details.
