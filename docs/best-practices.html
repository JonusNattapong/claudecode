Best Practices

Tips, patterns, and workflow recommendations for effective use of Clew.

Project Setup

Always Write a CLAUDE.md

A good CLAUDE.md is the single highest-leverage thing you can do. It teaches Clew your project's conventions, architecture, and rules before every session.

# CLAUDE.md
## Build
- Use `bun test` for tests, `bun run build` for production.
- This is an ESM project. Don't add CommonJS files.

## Conventions
- Components use React 19 + Ink 6.
- Use Zod for validation, not Yup.
- Files are kebab-case, functions are camelCase.

## Rules
- Don't add dependencies without strong justification.
- Prefer surgical edits over broad rewrites.

Use Project-Level Configuration

Create .claude/settings.json in your project root to set project-specific defaults:

{
  "model": "claude-sonnet-4-20250514",
  "permissions": {
    "allow": [
      "Bash(git *)",
      "Bash(bun test *)",
      "Bash(bun run build)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)"
    ]
  }
}

Prompting

Be Specific About Scope

Vague prompts get vague results. Name files, functions, and constraints:

Instead ofTry
"Fix the bug""Fix the null reference in src/utils/parser.ts:142 — it should return an empty array instead"
"Add auth""Add JWT authentication middleware to src/api/. Use the existing pattern from authMiddleware.ts"
"Refactor this""Extract the validation logic from UserForm.tsx into src/utils/validators.ts without changing behavior"

Use Plan Mode for Complex Changes

For multi-file refactors or new features, switch to plan mode first:

/permissions plan
# Now Clew will present an implementation plan before writing code.
# Review the plan, then approve or refine.

Use /compact Proactively

Don't wait for auto-compaction. If you've been in a long session with many tool calls, run /compact to keep the conversation focused and reduce token costs.

Switch Models Mid-Session

Use /model to switch providers or models without restarting. Press d in the picker to set a new default. Recent models are tracked and shown at the top.

Tool & Permission Strategy

Layer Your Permissions

Start strict and relax as you build trust:

  1. New project: default mode — get prompted for each sensitive action.
  2. After adding CLAUDE.md: auto mode with allow rules for common operations.
  3. Trusted workspace: acceptEdits for rapid file editing iteration.
  4. CI/automation: bypassPermissions only in isolated sandboxes.

Create Targeted Allow Rules

Prefer specific allow rules over blanket bypass. Rules are auditable and scoped:

# Good: specific and reviewable
Bash(git diff *)
Bash(git log *)
Bash(bun test *)
Bash(bun run build)

# Avoid: too broad
Bash(*)

Agent & Automation Workflows

Delegate to Sub-agents for Parallel Work

For tasks that can be split, use sub-agents to work in parallel:

> "Spawn a sub-agent to run the test suite while I review the diff.
   Then spawn another to check for unused imports."

Use Scheduled Tasks for Recurring Work

Set up recurring checks that run even when you're away:

/task           # Interactive form
# → Name: "PR Monitor"
# → Schedule: every-minutes / 30
# → Prompt: "Check for new PR comments and summarize any that need my attention"

Use /loop for Quick Recurring Commands

/loop 5m /babysit-prs        # Monitor PRs every 5 minutes
/loop 1h /status               # Health check every hour

Context Window Management

Monitor Context Usage

Keep an eye on the status bar context percentage. Use /context to see a detailed breakdown of token distribution across System Prompt, Tools, Rules, Chat, and Sub-agents.

Keep Sessions Focused

  • Start a new session for unrelated tasks (use /resume to switch).
  • Use /clear to reset context when changing topics.
  • Use /resume <N> to resume with only the last N exchanges.

Plugin & Extension Workflow

Audit Plugin Costs

Before installing a plugin, check its token impact:

/plugin browse          # Browse marketplace
/plugin-details <name>   # Check token cost per session

Keep Plugins Updated

/plugin update          # Update all plugins
/plugin update <name>   # Update specific plugin
/reload-plugins          # Reload without restart

Troubleshooting

When Things Go Wrong

  1. Run /doctor — Diagnoses environment, API keys, MCP servers, and common issues.
  2. Check /status — Shows model, provider, API status, and context usage.
  3. Enable debug logging — DEBUG=1 clew or clew --debug.
  4. Provider-specific debug — DEBUG=provider:anthropic clew.
  5. Check /mcp list — MCP config parse errors are displayed here.

Recovering from Stuck Sessions

  • Ctrl+C — Interrupt the current operation.
  • Ctrl+Z then resume — Suspends the session for later recovery.
  • /compact — If the model is struggling with context overflow.
  • clew --resume — Pick up from a previous session checkpoint.

Performance Tips

  • Use .gitignore-aware search — Clew's Glob and Grep tools respect .gitignore. Keep it updated.
  • Prefer targeted tests — "run the test in src/utils/parser.test.ts" is faster than "run all tests".
  • Use NO_FLICKER=1 — On slower terminals, this provides smoother rendering.
  • Use WSL2 on Windows — Best experience for Windows users.
