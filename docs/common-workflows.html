Common Workflows

Step-by-step guides for everyday tasks

Starting a New Project

  1. Launch Claudevil in your desired directory: claude
  2. Ask Claude to scaffold the project: "Create a new React app with Vite"
  3. Run /init to generate a CLAUDE.md with project-specific instructions
  4. Use /model to switch providers if needed

Debugging a Bug

  1. Describe the bug clearly — include error messages, expected vs. actual behavior
  2. Let Claude explore the codebase and identify the root cause
  3. Review the proposed fix before applying
  4. Run /rewind if the fix needs to be undone
  5. Run tests to verify: bun test <path>
Tip For complex debugging, provide reproduction steps and any relevant logs. Claude works best with evidence, not guesses.

Code Review

  1. Stage your changes and create a commit
  2. Ask: "Review the latest commit for bugs"
  3. Claude will inspect the diff, check for correctness, security issues, and test gaps
  4. Apply suggestions with --fix or discuss findings

Refactoring Code

  1. Be specific about the goal: "Extract the validation logic from this function into a separate module"
  2. Set a clear boundary — what files should change
  3. Review the diff before accepting changes
  4. Run the type checker: bun x tsc --noEmit
  5. Run tests to confirm nothing broke: bun test

Adding a Feature

  1. Describe the feature: "Add a --json flag to the /status command"
  2. Let Claude explore the codebase for the relevant command handler
  3. Review the implementation plan before code is written
  4. Test the feature manually and with bun test <path>
  5. Run /rewind if the feature needs to be rolled back

Using Multiple Providers

  1. Set your API keys as environment variables (see Environment Variables)
  2. Run /model to select a provider and model at runtime
  3. Switch freely between Anthropic, OpenAI, Google, DeepSeek, Groq, and others
  4. Claudevil's adapter layer normalizes the experience across providers

Working with Plugins

  1. Browse available plugins: /plugin discover
  2. Install a plugin: /plugin install <plugin-id>
  3. List installed plugins: /plugin list
  4. Remove a plugin: /plugin remove <plugin-id>
  5. See Plugin Development to create your own

Using MCP Servers

  1. Add an MCP server to your .claude/settings.json or install one via a plugin
  2. Claude automatically connects and makes the server's tools available
  3. Manage servers at runtime: /mcp list, /mcp restart

Voice Dictation

  1. Install SoX (brew install sox / apt install sox)
  2. Authenticate: /login
  3. Enable: /voice
  4. Hold Space while speaking, release to submit

Remote Collaboration (Bridge)

  1. Start a bridge session: /bridge start
  2. Share the generated URL with a collaborator
  3. Both parties can see and interact with the same session
  4. Stop the bridge: /bridge stop

Saving & Restoring Context

  1. Save current session context: claude session save "description"
  2. List saved sessions: claude session list
  3. Restore a session later: claude session restore
  4. Use /compact when context becomes large during a session

Checking System Health

  1. Run /doctor for a full diagnostic — checks provider connectivity, plugin state, hooks, and configuration
  2. Run /status to see current model, provider, and context usage
  3. Run /context to inspect the context window breakdown
