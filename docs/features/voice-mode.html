Voice Mode

Hands-free voice dictation with push-to-talk

TL;DR Run /voice to toggle voice dictation on/off. Hold the configured shortcut key while speaking (default: Space). Requires a microphone and audio recording tools.

Overview

Voice mode converts speech-to-text in real time using a speech recognition backend. It is designed for hands-free prompt entry — hold the push-to-talk key, speak naturally, and release to submit the transcribed text as your prompt. Audio recording stops when you release the key, so there is no manual trimming or editing step.

Requirements

RequirementDetails
Audio recording toolSoX (cross-platform) or arecord (Linux ALSA). Install SoX via your package manager (brew install sox, apt install sox, or sourceforge.net).
MicrophoneSystem default input device. Grant microphone permission when prompted by your OS.
Speech-to-text backendClaude.ai OAuth (recommended, via /login) or a compatible API key (OPENAI_API_KEY, GROQ_API_KEY, or a self-hosted Whisper endpoint).

Quick Start

1. Install audio tools

# macOS
brew install sox

# Linux (Debian/Ubuntu)
sudo apt install sox

# Windows
Download installer from https://sourceforge.net/projects/sox/

2. Authenticate

# Log in with your Claude.ai account (required for the default backend)
claude /login
Alternative backends Voice mode also works with Whisper-compatible APIs. Set OPENAI_API_KEY, GROQ_API_KEY, or WHISPER_API_KEY + WHISPER_API_URL to use an alternative STT backend without OAuth.

3. Enable voice

claude /voice

# Output
Voice mode enabled. Hold Space to record.

Toggle off at any time by running /voice again. Use /config to change the dictation language or adjust the push-to-talk keybinding.

Command Reference

CommandDescription
/voiceToggle voice dictation on or off. Runs pre-flight checks (microphone, backend, dependencies) before enabling.
/loginSign in with Claude.ai to enable the default OAuth-based voice backend.
/configChange dictation language and other voice-related settings.

Keybinding

The default push-to-talk key is Space (hold to record, release to submit). You can rebind it in your settings via the voice:pushToTalk keybinding: 

{
  "keybindings": {
    "voice:pushToTalk": {
      "key": "v",
      "mod": "mod"
    }
  }
}

Architecture

Microphone → SoX/arecord → PCM audio → STT backend → transcribed text → PromptInput

Backends:
  ├─ Claude.ai voice_stream (OAuth, default)
  ├─ OpenAI Whisper API (OPENAI_API_KEY)
  ├─ Groq Whisper API (GROQ_API_KEY)
  └─ Custom Whisper endpoint (WHISPER_API_KEY + WHISPER_API_URL)

Guard:
  isVoiceModeEnabled() = (hasVoiceAuth() || hasAlternativeSttKey()) && isVoiceGrowthBookEnabled()

Troubleshooting

No audio recording tool found

Install SoX (see Quick Start above). On Linux, arecord from the alsa-utils package is also supported.

Microphone access denied

Grant mic permission in your OS settings then run /voice again:

  • macOS: System Settings → Privacy & Security → Microphone
  • Windows: Settings → Privacy → Microphone
  • Linux: Your system's audio settings (PulseAudio/PipeWire)

Voice mode is not available

You need either a valid Claude.ai OAuth session (/login) or an alternative STT API key set in your environment.
