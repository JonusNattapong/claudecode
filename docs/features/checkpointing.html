Checkpointing

Automatic file snapshots and conversation rewind

TL;DR Claudevil automatically snapshots files after every assistant turn. Use /rewind (alias: /checkpoint, /undo) to open a visual timeline and restore files or the conversation to any prior state.

Overview

Checkpointing tracks every file change made during your session. After each assistant message, a snapshot is taken of all files modified up to that point. The /rewind command opens a state selector that lets you browse these snapshots, preview the diff between states, and restore either the file contents, the conversation history, or both to an earlier point.

How It Works

  1. Capture — After each assistant turn, fileHistoryMakeSnapshot records the current state of every modified file into a compressed snapshot.
  2. Diff — When you open the rewind UI, fileHistoryGetDiffStats computes the per-file diff between the current state and the target snapshot.
  3. Restore — Confirming a rewind replays the inverse of each edit, restoring files to their state at that point. The conversation is also trimmed to match.

Command Reference

CommandDescription

/undoAlias for /rewind.

Using /rewind

When you run /rewind, a message selector opens showing your conversation history with snapshot indicators. Navigate with the keyboard, select a message, and the UI displays:

  • Changed files — which files differ from the current state
  • Diff preview — per-file unified diff showing what would be restored
  • Scope toggle — restore files only, conversation only, or both

Architecture

Assistant turn completes
  → fileHistoryMakeSnapshot()
     → reads current file contents
     → compresses into snapshot blob
     → stores in session log (fileHistorySnapshots)

User runs /rewind
  → fileHistoryGetDiffStats(state, messageId)
     → decompresses target snapshot
     → diffs against current files
     → shows preview in selector

User confirms
  → fileHistoryRewind(state, messageId)
     → for each changed file:
        restore from snapshot content
     → truncates conversation log
     → updates UI state
