Model Management

Claude Code provides a unified model management system across all 15 AI providers. Switch models at runtime, discover available models, and configure per-session defaults.

Listing Available Models

All Models

/model list

Shows all available models across all configured providers. The model list is cached for 5 minutes to avoid excessive API calls.

Provider-Specific Models

/model list anthropic   # Anthropic Claude models
/model list openai      # OpenAI GPT models
/model list google      # Google Gemini models
/model list openrouter  # OpenRouter models (100+)

Refresh Model Cache

/model refresh

Forces an immediate refresh of the model list from all configured providers, bypassing the 5-minute cache.

Switching Models

Direct Model Selection

/model claude-sonnet-4-7
/model openai/gpt-5.5
/model google/gemini-3.1-pro

When a provider prefix is included (e.g., openai/), the provider is switched automatically along with the model.

Interactive Model Picker

Press Meta+P (or Alt+P on some terminals) to open the interactive model picker. Navigate with arrow keys and press Enter to select.

Via Configuration

{
  "model": "claude-sonnet-4-7",
  "provider": "anthropic"
}

Reasoning Effort Control

Claude models support reasoning effort levels that control how much thinking the model does before responding:

  • Low — Minimal reasoning, fastest responses
  • Medium — Balanced reasoning (default)
  • High — Extended reasoning for complex tasks

Adjust effort via the model picker using Left / Right arrow keys, or toggle thinking mode with Meta+T / Ctrl+Shift+T.

Provider + Model Interaction

The provider and model selections interact as follows:

  • Provider-first: Use /provider openai then /model gpt-5.5. The model is resolved within the current provider.
  • Model-first: Use /model openai/gpt-5.5. Both provider and model switch simultaneously.
  • Default model: Each provider has a defaultModel (e.g., claude-sonnet-4-7 for Anthropic). Switching providers without specifying a model uses this default.

Model Registry

The model registry (src/services/ai/providerRegistry.ts) is the single source of truth for all available models. It defines:

  • Model IDs and human-readable labels
  • Capability profiles (tool calling, vision, streaming, context size)
  • Optional tags for filtering and discovery
  • Supported API types (messages, responses)

Fast Mode

Toggle fast mode with Meta+O or use:

/fast

Fast mode uses the same model with optimized output generation for lower latency. Not all providers support fast mode — it is primarily available for Anthropic Claude models.

Model Caching: Available models are cached for 5 minutes per provider. The cache is automatically invalidated when you switch providers or explicitly run /model refresh. Model discovery requires valid API credentials for each provider.
