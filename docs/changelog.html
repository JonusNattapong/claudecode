Changelog

All notable changes to Clew, following a practical changelog format.

Categories: Added (new capabilities), Changed (behavior/UX), Fixed (bug fixes), Security (permission/sandbox/auth hardening), Internal (tests/types/refactors), Removed (deprecated features).

2.1.179 — 2026-05-28

Added

  • Npm global install auto-update notice — One-time notice when CLI detects npm global install lacks write permissions for auto-update. Notice resets after a successful update. Shows remediation link and suggests native installation.
  • /doctor last update attempt — Doctor screen now displays the result of the most recent update attempt (status, channel, version, error, install type) with human-readable age and recovery hints.
  • Unified release channel resolver — resolveReleaseChannel() is the single source of truth for channel resolution across all update paths (CLI, auto-updater, native installer).
  • COLUMNS/LINES in status line env — Status line commands now receive terminal dimensions, allowing scripts to adapt output width.
  • MCP progress in collapsed tool view — Collapsed tool rows display MCP progress notifications (message or percentage) alongside the MCP server name.
  • Agent view dispatch autocomplete — Typing / in agent dispatch input shows autocomplete from slash commands, skills, and plugins with source labels ([cmd], [bundled], [skill], [plugin]).
  • Agent view skill dispatch — Dispatch input recognizes /skill-name syntax. Project skills take priority when names collide.
  • skipLfs for plugin marketplace sources — GitHub and Git marketplace sources support skipLfs: true to skip Git LFS downloads during clone/pull.

Changed

  • Narrow terminal help wrapping — --help output uses actual terminal width (minimum 60 columns).
  • Responsive help menu — PromptInputHelpMenu switches to vertical stacked layout when terminal width is below 80 columns.
  • Agent view PR column — Shows PR #123 for single PR, 3 PRs for multiple.
  • Welcome frame tracks mascot body color — Welcome screen border uses the same color as the Clawd mascot body (customizable via /color → Mascot tab).

Fixed

  • Clickable file:// link normalization — Added normalizeFileUrl() that fixes malformed file:// URLs (Windows paths, missing third slash, spaces).
  • Remote Control zombie session cleanup — Agent view marks sessions as detached on unmount and runs periodic cleanup of stale remote-control sessions.
  • Windows installer recovery instructions — Native installer post-install verification includes platform-specific recovery hints.

2.1.158 — 2026-05-23

Added

  • GFM Task List Checkboxes — Markdown output renders - [ ] todo / - [x] done as checkboxes instead of plain bullets.
  • Diff keyboard scrolling — /diff detail supports PgUp/PgDn, Home/End, vim-style j/k, Space, and arrow keys.
  • /usage contributing factors — "What's contributing to your limits usage?" breakdown section from API data.
  • /recap command — New command for session recap functionality.
  • Agent view CLI — /agents opens interactive agent view with --cwd, --config/--manage options. Supports --json.

Changed

  • LogoV2 rebranding — Updated all logo components with refined visuals.
  • Agent dashboard redesign — Refactored to use design system components (Dialog, Tabs, ProgressBar, StatusIcon, Divider).
  • Voice module un-gated — Removed feature flag gate from /voice; always accessible.
  • PromptInput, StatusLine, Settings UI updates — Various UI refinements.
  • Permission mode updates — Refined permission mode logic and type definitions.

Fixed

  • PowerShell cd built-in permission bypass — cd.., cd\, cd~, and bare drive letters now correctly detected as cwd-changing.
  • Bash PWD/DIRSTACK stale variable tracking — After cd/pushd/popd, PWD, OLDPWD, and DIRSTACK are invalidated from variable tracking scope.
  • Jump-to-bottom pill dismissal — Pill now dismisses immediately instead of waiting for next scroll tick.
  • /voice native module crash on Windows — Wrapped audio-capture-napi in try/catch with Windows SoX fallback.

Removed

  • /eval command — Removed the verification harness and its entire src/eval/ framework.
  • /profile command — Removed profile module (profile command + profileManager utility).

2.1.155 — 2026-05-22

Added

  • /autofix-pr — Launch remote Clew on web to fix CI errors and address review comments on a GitHub PR.
  • /loop — Run a prompt or slash command on a recurring interval. Accepts s/m/h/d suffixes and natural language "every N minutes" clauses.
  • Interactive Argument Ghost Text — Command parameter hints render as inline ghost text. Tab or Right Arrow to accept and fill the placeholder.
  • /plugin-details / /plugin-info — Show component inventory and estimated per-session token cost for any installed plugin.

Changed

  • Remote Control enabled by default — All sessions are remote-control-ready without explicit config.
  • /autofix-pr available to all users — Moved from INTERNAL_ONLY_COMMANDS to main COMMANDS array.
  • /loop skill un-gated — Removed feature flag requirement from loop skill and underlying cron tools.

2.1.153 — 2026-05-21

Security

  • Prompt injection boundary — Worker task prompts wrap user-controlled descriptions in <policy> + <task_data> XML tags with explicit system override instructions. sanitizeForXml() strips control characters and CDATA closure injection.

Added

  • Task lease/lock system — Prevents duplicate task execution with lease ownership, release, and expiry.
  • Retry dead-letter & backoff — New dead_letter status with exponential backoff and auto-dead-letter after max retries.
  • Project namespace — projectRoot field ensures workers spawn in correct project directory.
  • File watcher debounce — 300ms debounce with self-write suppression.
  • Graceful shutdown — SIGTERM stops accepting tasks → releases leases → kills workers → closes watcher → flushes queue → exits.

Changed

  • Auto-start opt-in — autoStart defaults false. Only explicit /daemon start enables it.
  • /daemon status — Shows autoStart, dead-lettered, and last error fields.
  • /task show — Shows projectRoot, lastError, deadLetterReason, leaseOwner, retryAfter.

2.1.152 — 2026-05-21

Added

  • /daemon command — start, stop, status, restart subcommands.
  • /task command — add, list, show, done, cancel, fail, retry, remove subcommands.
  • Supervisor IPC — autonomous_start, autonomous_stop, autonomous_status commands.

2.1.146 — 2026-05-19

2.1.145 — 2026-05-19

Added

  • KiloCompact — Local context compaction engine that collapses verbose directory trees, glob outputs, stack traces, and large text dumps. Consolidates redundant failed tool runs. Semantic turn pruning with protected early/recent turns.
  • /resume <N> — Numeric argument resumes the latest conversation with only the last N exchanges.
  • --resume <N> CLI support — Resumes latest conversation with limited history window.
  • Recent model tracking — Recently used models stored in recentModels setting (capped at 5), surfaced near top of model picker.
  • Background session improvements — ps, stop, attach, --bg work at runtime. --name support. Elapsed duration on completion.
  • Stream-stall recovery — Pre-response stream stalls retry streaming once before falling back to non-streaming mode.
  • /usage-credits command — Keeps /extra-usage as alias.

Changed

  • /model selection applies to current session; press d to set default.
  • Resumed sessions preserve the model they were using through sessionModel transcript field.
  • Statusline context bar freezes to last known non-zero usage instead of dropping to 0% during streaming.
  • Permission-mode cycling: bypassPermissions no longer immediately follows auto.

Fixed

  • Startup hangs from slow side-channel API calls (now timeout after 15s).
  • MCP tools/list pagination with nextCursor.
  • MCP SVG image handling — saved to disk instead of crashing.
  • MCP config error visibility in mcp list.
  • Grep-style command exit handling — egrep, fgrep, git grep, git diff exit code 1 no longer treated as hard failure.
  • Read-before-edit checks for head and tail.
  • Image extension mismatch — fallback to text handling.
  • Skill file watcher file-descriptor exhaustion.
  • Skill usage in headless/non-interactive mode.
  • Side-query model fallback with custom ANTHROPIC_BASE_URL.
  • Plugin cache-miss errors now show actionable install hint.
  • Windows background commands (ps, stop, attach, --bg).
  • /branch after worktree entry.
  • Ctrl+C in ! commands now kills running process correctly.
  • Scrolling in attached background sessions (PgUp/PgDn).

Security

  • Improved permission-mode cycling to reduce accidental bypass-mode transitions.
  • Improved background-mode rejection messages for unsafe execution gates.
