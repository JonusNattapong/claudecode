Scheduled Tasks

Schedule recurring and one-shot tasks with /task, /loop, and cron-based scheduling.

Quick Reference

CommandUse For
/taskInteractive scheduled task creation form. Primary user-facing workflow.
/loop <interval> <command>Run a command on a recurring interval (e.g. /loop 5m /babysit-prs).
/task addAdd a one-shot or recurring task to the queue.
/task listList all pending and scheduled tasks.
/task scheduledOpen the scheduled task form explicitly.

/task — Interactive Form

Running /task with no subcommand opens a form for creating scheduled tasks:

  1. Name — Human-readable task label.
  2. Schedule type — daily, weekdays, hourly, every-minutes, one-shot, or custom (5-field cron).
  3. Time / cron value — Depends on schedule type.
  4. Prompt — The instruction to execute when the task fires.
  5. Storage mode — Session-scoped (runtime only) or file-backed (persists across restarts).

/loop — Recurring Command Runner

Run any prompt or slash command on a recurring interval:

/loop 5m /babysit-prs        # Check PRs every 5 minutes
/loop 1h /status               # Health check every hour
/loop 30s "check the build"    # Check build every 30 seconds
/loop every 10 minutes /task list

Interval suffixes: s (seconds), m (minutes), h (hours), d (days). Also parses natural language "every N minutes" clauses from the prompt.

Cron Scheduling

Tasks are stored in .claude/scheduled_tasks.json using a 5-field cron format:

┌─────────── minute (0–59)
│ ┌─────────── hour (0–23)
│ │ ┌─────────── day of month (1–31)
│ │ │ ┌─────────── month (1–12)
│ │ │ │ ┌─────────── day of week (0–6, Sun–Sat)
│ │ │ │ │
* * * * *

Examples

CronSchedule
0 9 * * 1-5Every weekday at 9:00 AM
*/15 * * * *Every 15 minutes
0 0 1 * *Midnight on the 1st of every month
30 14 27 2 *Feb 27 at 2:30 PM (pinned, once per year)

CronTask Schema

{
  "id": "task-abc123",
  "cron": "0 9 * * 1-5",
  "prompt": "Summarize yesterday's commits",
  "createdAt": 1716854400000,
  "recurring": true,
  "lastFiredAt": 1716940800000
}
FieldDescription
idUnique task identifier.
cron5-field cron string in local time.
promptPrompt to enqueue when the task fires.
createdAtEpoch ms when the task was created. Anchor for missed-task detection.
lastFiredAtEpoch ms of most recent fire. Preserves next-fire computation across restarts.
recurringWhen true, reschedules after firing instead of being deleted.
permanentExempt from auto-expiry. Only set by system tasks, not via CronCreateTool.

Missed Task Detection

When Clew starts, the scheduler detects one-shot tasks that should have fired while it wasn't running. Missed tasks are reported via a notification; they are not auto-executed — the AI must ask for confirmation first.

Task Queue Management

/task add "review open PRs" --priority high
/task list                    # List all tasks
/task show <id>               # Show details including projectRoot, lastError, lease info
/task done <id>               # Mark as completed
/task fail <id>               # Mark as failed
/task retry <id>              # Retry a failed task
/task requeue <id>            # Re-queue a dead-lettered task
/task cancel <id>             # Cancel a pending task
/task remove <id>             # Remove a task permanently

Dead Letter & Retry

Tasks that exhaust their retry count move to dead_letter status:

  • Exponential backoff — base × 2^count, capped at 1 hour.
  • Auto-recovery — Expired leases auto-recover tasks to pending state.
  • Manual requeue — Use /task requeue <id> to move a dead-lettered task back to pending.

Programmatic Access

Model-facing cron tools provide programmatic access to the scheduler:
