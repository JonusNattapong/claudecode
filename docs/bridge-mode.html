Bridge Mode

How Claude Code enables remote collaboration through shared session URLs.

What is Bridge Mode?

Bridge Mode allows users to share their Claude Code session with others via a URL. Remote users can view and control the session.

Bridge Files

src/bridge/
├── bridgeMain.ts              # Main bridge logic
├── bridgeConfig.ts            # Bridge configuration
├── bridgeEnabled.ts           # Enable/disable
├── bridgeUI.ts                # Bridge UI components
├── bridgeApi.ts               # API endpoints
├── bridgeMessaging.ts         # Message handling
├── replBridge.ts              # REPL bridge
├── remoteBridgeCore.ts        # Remote session handling
├── sessionRunner.ts           # Session execution
├── createSession.ts           # Session creation
├── bridgePermissions.ts       # Permission callbacks
├── trustedDevice.ts           # Device trust
├── workSecret.ts              # Work secrets
├── jwtUtils.ts                # JWT utilities
└── inboundMessages.ts         # Inbound message handling

Starting Bridge

// Start bridge mode
/bridge

// Or via command line
claude --bridge

How It Works

// 1. User starts bridge mode
await startBridge()

// 2. Generate shareable URL
const bridgeUrl = generateBridgeUrl(sessionId)

// 3. Remote user opens URL
// 4. WebSocket connection established
// 5. Messages relayed between sessions
// 6. Both users see same UI
// 7. Either can take control

Session URL

// Bridge session URL format
https://claude.ai/bridge/{sessionId}?token={accessToken}

// Or local bridge
http://localhost:PORT/bridge/{sessionId}

Key Learnings

  1. WebSocket-based — Real-time message sync
  2. Shareable URLs — Easy to share sessions
  3. State synchronization — Both users see same state
  4. Permission control — Can restrict remote actions
  5. Device trust — Trusted devices get more access
  6. JWT authentication — Secure token-based auth
