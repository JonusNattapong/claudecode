CLI Reference

Flags, options, and usage patterns for the clew command.

Usage

clew [flags] [prompt]

# Run interactively
clew

# Run with an initial prompt
clew "explain this project"

# Pipe input from another command
cat errors.log | clew -p "summarize these errors"

Flags

FlagDescription
-p, --print <prompt>Non-interactive print mode. Pass a prompt directly or pipe input via stdin
-c, --continueContinue the most recent session
-r, --resumeShow picker to resume a saved session
--model <name>Set the model at startup (e.g. --model deepseek-v4-pro)
--provider <name>Set the provider at startup (e.g. --provider openai)
--debugEnable debug logging
--verboseEnable verbose output
--no-autoupdateDisable auto-update checks
--from-pr <url>Resume the session that created a given PR
--output-format <fmt>Output format for print mode: text (default), json, markdown
--skip-permission-checkSkip initial permission system verification
-v, --versionShow version and exit
--helpShow help and exit

Print Mode

Non-interactive mode for scripting, CI/CD, and piping:

# Simple query
clew -p "find all unused exports in src/"

# Pipe from stdin
cat test-output.log | clew -p "find failures and suggest fixes"

# Structured output
clew -p --output-format json "list all dependencies"

# Pipe to jq for processing
clew -p "list all TypeScript errors" | jq '.errors'

Remote & Bridge

FlagDescription
--teleportPull a remote session from claude.ai/code into your local terminal
--bridgeStart in bridge mode for remote collaboration
# Pull a cloud session to your terminal
clew --teleport

# Start bridge mode
clew --bridge
