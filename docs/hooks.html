Hooks

Automate workflows by running shell commands before and after Clew actions.

Hooks let you run shell commands automatically at key points in the agent lifecycle. They're configured in ~/.claude/settings.json or .claude/settings.json at the project level.

Use Cases Auto-format code after every file edit, run lint before a commit, send notifications when a long task finishes, validate commands before execution, or enforce project rules.

Hook Events

EventWhen It FiresTypical Use
PreToolUseBefore any tool executesValidate input, block dangerous operations
PostToolUseAfter any tool completesAuto-format, run tests, send notifications
PreBashBefore a shell command runsAudit commands, enforce allow/deny lists
PostBashAfter a shell command completesLog output, flag errors
PrePromptBefore sending a prompt to the modelInject context, modify the prompt
StopWhen a session endsCleanup, save summaries

Configuration

Hooks are defined in the hooks array of your settings file. Each hook specifies an event, a shell command, and optional filters.

{
  "hooks": [
    {
      "matcher": {
        "type": "PostToolUse",
        "tool": "FileEditTool"
      },
      "command": "bun run format"
    },
    {
      "matcher": {
        "type": "PostToolUse",
        "tool": "FileWriteTool"
      },
      "command": "bun run check --write src/"
    },
    {
      "matcher": {
        "type": "PreBash"
      },
      "command": "echo 'Running: $CLEW_COMMAND'"
    }
  ]
}

Matcher Configuration

FieldTypeDescription
typestringHook event name (see table above)
toolstring(Optional) Filter by tool name, e.g. FileEditTool, BashTool
globstring(Optional) Filter by file glob pattern, e.g. src/**/*.ts
Environment Variables Hooks receive context via environment variables: $CLEW_TOOL, $CLEW_COMMAND, $CLEW_FILE, and $CLEW_RESULT.

Async & HTTP Hooks

Hooks can also be HTTP endpoints instead of shell commands. Use "url" + optional "method" in the hook config.

{
  "hooks": [
    {
      "matcher": { "type": "PostToolUse" },
      "url": "https://hooks.example.com/clew",
      "method": "POST"
    }
  ]
}

Prompt Hooks

Prompt hooks (PrePrompt) receive the assembled prompt on stdin. The command can read, modify, and output a transformed prompt.

{
  "hooks": [
    {
      "matcher": { "type": "PrePrompt" },
      "command": "node scripts/inject-context.js"
    }
  ]
}

Best Practices

  • Keep hooks fast — slow hooks delay the agent loop. Aim for sub-second execution.
  • Use project-level hooks — .claude/settings.json is shared with your team via git.
  • Use global hooks — ~/.claude/settings.json for personal automation that applies everywhere.
  • Filter by tool — avoid running format on every tool; scope to FileEditTool and FileWriteTool.
  • Test hooks independently — run the command manually before adding it as a hook.

Debugging Hooks

Use /doctor to check if your hooks are loaded correctly. Hook output is visible in debug mode:

DEBUG=1 clew
