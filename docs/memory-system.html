Memory System

How Claude Code remembers user preferences and project context across sessions.

Memory Types

TypeScopeDescription
userprivateUser's role, goals, preferences
feedbackprivate/teamGuidance from user about how to work
projectteamProject-specific conventions
referenceteamExternal references, docs

Memory Files

.claude/
└── memory/
    ├── MEMORY.md           # Main entry point (auto-loaded)
    ├── user/              # User memories
    │   └── *.md
    ├── feedback/           # Feedback memories
    │   └── *.md
    ├── project/            # Project memories
    │   └── *.md
    └── reference/          # Reference memories
        └── *.md

How It Works

1. Loading

src/memdir/memdir.ts

At session start, MEMORY.md is loaded into system prompt:

// Load memory prompt for system prompt
await loadMemoryPrompt()

// Returns formatted memory content
// Included in system prompt automatically

2. Finding Relevant Memories

src/memdir/findRelevantMemories.ts

AI can find relevant memories based on context:

// Search memory files for relevant content
const memories = await findRelevantMemories(query)

3. Memory Detection

src/utils/memoryFileDetection.ts

Detects if file operations target memory:

isMemoryFile(path): boolean      // Is this a memory file?
isMemoryDirectory(path): boolean // Is this memory dir?
isMemorySearch(toolInput): boolean  // Is search targeting memory?

4. Memory Command

src/commands/memory/

User can manage memories via:

/memory              # Open memory manager
/memory write       # Write new memory
/memory search      # Search memories

Memory Frontmatter

Each memory file can have frontmatter:

---
type: user
owner: private
---

# My Memory Title

Memory content here...

Auto Memory

Claude Code can automatically remember important context:

// settings.json
{
  "autoMemoryEnabled": true,
  "autoMemoryDirectory": "~/.claude/projects/myproject/memory/"
}

Context Compaction

When context gets too large, memories help preserve important info:

src/services/compact/
├── compact.ts            # Main compaction
├── sessionMemoryCompact.ts # Compact session
├── autoCompact.ts        # Auto-compact
└── microCompact.ts      # Incremental compact

Key Learnings

  1. Memory is separate from context — Loaded at session start, not part of rolling window
  2. Type-based organization — user, feedback, project, reference
  3. Scoped access — private vs team memories
  4. Frontmatter metadata — type, owner, tags
  5. Auto-loading — MEMORY.md loaded automatically
  6. Detection — Can detect memory file operations for special handling
Pattern: Memory system is a hybrid approach — explicit files (user controls) + implicit loading (automatic at session start).
